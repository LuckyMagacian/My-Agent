# 项目文档目录结构规范

## 目录结构

```
docs/
├── requirements/              # 需求文档
│   └── {YYYY-MM-DD}-{主题}-需求说明书.md
├── design/                    # 设计文档
│   ├── {YYYY-MM-DD}-{主题}-设计说明书.md       # 概要设计
│   └── {YYYY-MM-DD}-{主题}-{模块}-详细设计.md  # 详细设计（关联概要设计）
├── changes/                   # 变更计划
│   └── {YYYY-MM-DD}-{主题}.md
├── bugs/                      # 问题修复
│   └── {YYYY-MM-DD}-{问题摘要}.md
├── analysis/                  # 分析报告
│   └── {模块}-分析报告.md
├── solution-optimization/     # 方案优化
│   └── {YYYY-MM-DD}-{主题}.md
└── 需求vs变更-区分指南.md      # 场景区分规则
```

## 需求 vs 变更 - 快速区分

| 判断维度 | 需求开发 | 功能变更 |
|---------|---------|---------|
| **代码状态** | 从零开始 | 已有代码基础 |
| **模块性质** | 新增独立模块 | 修改现有模块 |
| **文档路径** | `docs/requirements/` | `docs/changes/` |
| **判断口诀** | 新模块、新架构、无历史 | 改现有、小范围、需兼容 |

**详细区分规则** → 参见 [需求vs变更-区分指南.md](需求vs变更-区分指南.md)

## 命名规范

### 日期格式
- 格式：`YYYY-MM-DD`（如 `2026-06-23`）
- 使用任务发起日期

### 主题命名
- 使用中文或英文，保持简洁
- 多个词用 `-` 连接
- 示例：`用户登录优化`、`api-refactor`

### 文件名示例
```
requirements/2026-06-23-用户登录优化-需求说明书.md
design/2026-06-23-用户登录优化-设计说明书.md
design/2026-06-23-用户登录优化-auth-详细设计.md
design/2026-06-23-用户登录优化-api-详细设计.md
changes/2026-06-20-接口字段调整.md
bugs/2026-06-22-登录超时问题.md
analysis/auth-分析报告.md
solution-optimization/2026-06-23-文档位置规范优化.md
```

**命名关联规则**：
- 同一主题的概要设计、详细设计使用**相同日期和主题前缀**
- 详细设计追加模块名：`{日期}-{主题}-{模块}-详细设计.md`
- 示例：概要设计 `2026-06-23-用户登录优化-设计说明书.md` → 详细设计 `2026-06-23-用户登录优化-auth-详细设计.md`

## 使用指南

| 场景 | 文档类型 | 创建时机 | 确认要求 |
|------|----------|----------|----------|
| 新功能开发 | 需求说明书 | 需求澄清后 | ✅ 用户确认 |
| 设计阶段 | 设计说明书 | 需求确认后 | ✅ 用户确认 |
| 模块实现 | 详细设计 | 设计确认后 | ✅ 用户确认 |
| 功能修改 | 变更计划 | 影响评估后 | ✅ 用户确认 |
| Bug 修复 | 问题分析 | 根因定位后 | ✅ 用户确认 |
| 代码分析 | 分析报告 | 分析完成后 | - |
| 方案改进 | 方案优化 | 方案设计后 | ✅ 用户确认 |
